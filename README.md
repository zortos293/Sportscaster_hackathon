# Sportscaster Hackathon MVP

Live AI text-to-speech sports commentary for niche sports. Stats come from the [ESPN MCP Server](https://apify.com/mrbridge/espn-mcp-server) on Apify; commentary is generated by an LLM and spoken via TTS.

### Data flow (one commentary cycle)

1. **Poller** calls `espn_live_scoreboard` or `espn_game_summary` for a tracked `event_id`.
2. **Diff engine** compares new snapshot to last state (score, period, leaders, status).
3. If something meaningful changed, **CommentaryEngine** writes 1вҖ“2 sentences in broadcaster voice.
4. **TTS** converts text to audio bytes; backend pushes `{ text, audio_url }` over WebSocket.
5. Client plays AI audio over (or instead of) the video feed.

### MVP modes

| Mode | Video source | Stats source | Best for demo |
|------|--------------|--------------|---------------|
| **Audio-only** | None / static poster | ESPN MCP poll | Fastest hackathon win |
| **YouTube replay** | YouTube embed URL | ESPN MCP poll | Feels like вҖңwatching a gameвҖқ |
| **Local MP4** | File in `samples/` | ESPN MCP poll | Offline / controlled demo |
| **Twitch live** | Twitch embed | ESPN MCP poll | Niche live sports |
| **True sync** | YouTube + game clock | ESPN play-by-play | Phase 2 |

Start with **YouTube replay + ESPN stats**. You do not need live video rights for a compelling demo.

## Where to get video

### Recommended for MVP (no licensing headaches)

1. **YouTube full-game or highlight replays** вҖ” Embed with the IFrame API. Mute original audio; overlay AI commentary. Search for вҖңfull gameвҖқ for college/niche leagues (lacrosse, rugby, minor league baseball). Use `video_source.type: "youtube"` in session config.

2. **Local sample file** вҖ” Drop an MP4 in `samples/` and set `video_source.type: "file"`. Good for rehearsed demos.

3. **No video (audio broadcast)** вҖ” Many вҖңAI sportscasterвҖқ demos are audio-first with a scoreboard UI. Simplest path to вҖңliveвҖқ feel using ESPNвҖҷs ~60s refresh.

### Nice-to-have live options

4. **Twitch** вҖ” Strong for niche sports (college club, esports-adjacent, international leagues). Embed channel; pair with ESPN stats when the league is covered.

5. **User-supplied HLS URL** вҖ” If you have rights to a stream, pass `video_source.type: "hls"` and play with hls.js. Backend stays the same.

6. **ESPN Watch / league apps** вҖ” Usually DRM + geo-blocked; avoid for hackathon unless you have partnership.

### What not to rely on for MVP

- Scraping ESPN video streams (ToS, DRM, brittle).
- Expecting ESPN MCP to provide video вҖ” it only provides **structured stats text**.

## Quick start (recommended вҖ” Next.js only)

Everything runs in the **Next.js app**. No Python server needed for the demo.

```bash
cd web
npm install
cp .env.local.example .env.local   # add ELEVENLABS_API_KEY (required for voice)
npm run dev
```

Videos are symlinked from `samples/` вҶ’ `web/public/samples/`.

Open http://localhost:3000/dashboard вҶ’ pick a match вҶ’ **press play**.

### How sync works

1. On video load, `/api/timeline` fetches ESPN scoring plays and maps them to video timestamps.
2. As the video plays, `timeupdate` fires commentary when each score event is reached.
3. `/api/commentary` writes the broadcaster line (LLM or template).
4. `/api/tts` speaks it via ElevenLabs.

### Env vars (`web/.env.local`)

| Variable | Required | Purpose |
|----------|----------|---------|
| `ELEVENLABS_API_KEY` | Yes (for voice) | Text-to-speech |
| `OPENAI_API_KEY` | No | Better commentary lines |
| `NEXT_PUBLIC_CONVEX_URL` | No | Auth (empty = demo mode) |

## Quick start (legacy Python backend)

### Create a session

```bash
curl -X POST http://localhost:8000/sessions \
  -H 'Content-Type: application/json' \
  -d '{
    "sport": "basketball",
    "league": "mens-college-basketball",
    "event_id": "401706957",
    "poll_interval_seconds": 45,
    "video_source": {
      "type": "youtube",
      "url": "https://www.youtube.com/watch?v=EXAMPLE"
    }
  }'
```

### Connect WebSocket

```
ws://localhost:8000/ws/sessions/{session_id}
```

Events: `commentary` (text + optional audio base64), `game_state`, `heartbeat`.

## Next steps after MVP

- **Play-by-play sync**: use `espn_game_summary` scoring plays + wall-clock alignment to YouTube seek.
- **Multi-voice personas**: home/away/analyst via TTS voice IDs.
- **Niche leagues off ESPN**: add a second stats adapter (league API, manual JSON feed, or vision on stream).
- **Low-latency TTS**: streaming TTS (ElevenLabs, Cartesia) for sub-second speech start.
